Calendario Editoriale

Per funzionare con Supabase il file deve essere su un server https://. Usiamo GitHub Pages — gratuito e pronto in 5 minuti.

⚠️ Perché non funziona aprendo il file direttamente? Il browser blocca le chiamate a internet da file locali (file://) per sicurezza. Caricando il file su GitHub Pages ottieni un link https:// che funziona ovunque — PC, telefono, cliente.
1
Crea un account GitHub gratuito

Vai su github.com/signup e registrati. Se ce l'hai già salta questo step.

2
Crea un nuovo repository

Vai su github.com/new → chiama il repository calendario → spunta Public → clicca Create repository.

3
Carica questo file HTML

Nella pagina del repository clicca uploading an existing file → trascina il file calendario_editoriale.html → rinominalo index.html → clicca Commit changes.

4
Attiva GitHub Pages

Nel repository vai su Settings → Pages → sotto Branch seleziona main → clicca Save. Aspetta 1-2 minuti.

5
Apri il tuo link

Il calendario sarà disponibile su https://tuonome.github.io/calendario. Condividi questo link con il cliente — entrambi potete modificare!

Connetti Supabase

Inserisci le credenziali del tuo progetto Supabase. Le stesse credenziali vanno condivise con il cliente.

Settings → General → Project URL
Settings → API Keys → copia la Publishable key (quella che inizia con sb_publishable_)
